02Design tokens

These are the only tokens the Companies page uses. Declare once, at :root.

/* Paper & ink */
--paper:     #FDFDFD;   // page background
--paper-2:   #F6F5F1;   // subtle fill (search box, callouts)
--ink:       #111418;   // primary text
--ink-2:     #4A4F57;   // secondary text, italic body
--ink-3:     #8A8F97;   // mono meta, dividers overlay

/* Structure */
--line:      #E5E3DE;   // section borders, topbar
--line-2:    #EFEDE8;   // row dividers (hairline)

/* Brand */
--brand:     #1B2A44;   // oxblood navy — CTA links, emphasized digits
--accent:    #8B2E1F;   // USC oxblood — underline scribble, eyebrow accent

/* Type */
--serif:    'Instrument Serif', Georgia, serif;
--sans:     'Inter', system-ui, sans-serif;
--mono:     'JetBrains Mono', monospace;

Rules:

  • Never introduce new colors on this page. If you need a signal color, pick from the existing palette or add one to the token file globally.
  • No gradients. Flat fills only. Brand marks are solid discs.
  • No shadows on list rows or the hero. Depth comes from borders and type weight.
  • Italic serif is the voice of Scout. Sans is structure. Mono is metadata. Don't mix voices within a single sentence.

Where the copy comes from

  • Headline = rendered from the Scout ladder, R4-tier sentence (always aggregate, never names). See Scout Spec §04.
  • Detail = rendered from the Scout ladder, R1 → R5 cascade. The hero uses whichever rung fires. See Scout Spec §04.
  • Name + mark = from the company record. mark.color is the company's primary brand color; mark.letters is 1–2 chars (initial or monogram). Never render an empty square — if we don't have a real color, fall back to --brand with the initial.
  • Meta = sector · city from company record.
  • Never hardcode the Priya Desai line. It's output from the ladder. If the ladder returns R4 or R5, the detail paragraph renders those rungs' sentence — NOT a placeholder.

Row content — 2 lines

Line 1 — name + city, flex row with justify-content: space-between, align-items: baseline, margin-bottom: 6px.

Line 2 — italic scout sentence, its own line, wraps naturally. Serif italic 13px, --ink-2, line-height: 1.5. Digits inside the italic sentence switch to sans + --brand + 600 (visible numeric emphasis inside a serif italic voice).

  • No right-arrow icon on rows. Whole-row cursor: pointer; hover state bumps name color to --brand.
  • City is mono 9px uppercase. Same style as the hero meta — keeps the page visually quiet.
  • Scout sentence on its own line. Never inline-truncate. If it wraps to two lines that's fine; the row grows.
  • No dividers at the TOP of the first row or BOTTOM of the last. Only between rows. Visually this looks like a list, not a box.

The page has no internal state for company data. It's a pure render of these props. Scout sentence generation happens server-side (or in a background worker) using the spec in Scout Intro Sentence — Spec.html, and the result is hydrated into company.scout before the page renders.

10Accessibility

  • Every row is a link (<a> or button with role=link), not a div with onClick.
  • Focus visible on all interactive elements (rows, CTA, footer link/input, tabs).
  • Hero mark has aria-hidden="true" — it's decorative; the company name below it is the label.
  • Mono italic combinations must clear 4.5:1 contrast. Check --ink-3 on --paper — currently passes. Check --ink-2 italic serif at 12.5px — passes.
  • Tab order: Topbar → tabs → Ask Scout → Hero CTA → List rows top-to-bottom → Footer search.
  • Do not use color alone to indicate the hero vs list. The "01 · START HERE" eyebrow does the semantic work.

11What NOT to do

  • No card backgrounds on the hero, list, or footer. Content sits on paper.
  • No right-edge arrows on list rows. Cursor pointer is the affordance.
  • No "Best match" pill or badge. The hero's size + eyebrow is the signal.
  • No icons in the tab row. Text only, mono uppercase, in the topbar.
  • No search input in the default state. Only a text link. The link expands into an input only when clicked.
  • Never render an empty mark (blank white square). Always have a letter + a color. Monogram fallback uses --brand.
  • Do not animate the hero entry. Page loads, content is there. No fade-in, no slide-up.
