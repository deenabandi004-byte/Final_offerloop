01 — Audit

What the current system is doing — and where it's fighting itself.

The product is solid; the surface is fighting you. The stack has two serifs, four "blues", both warm-cream and cool-navy tokens, and CSS that uses !important to resolve its own conflicts. The result feels both too quiet (cream-on-cream cards) and too loud (navy sidebar shouting next to a pastel content area).

High · Identity
Two competing palettes

Warm (#FEFDFB / #FAF9F6) and cool (#FAFBFF / #E2E8F0) tokens both live in :root. The app alternates between them — navy sidebar + cool cards + warm body — and nothing feels intentional.

High · Type
Lora + Inter, applied inconsistently

Lora is used for every heading; the landing page also loads Instrument Serif, DM Sans, Libre Baskerville, and IBM Plex Mono. Five families, no clear roles. Headings read as "blog post", not "product".

High · Color
Brand blue is used as wallpaper

#3B82F6 appears as tab underline, primary button, nav highlight, CTA, focus ring, link, chart color, and progress bar — simultaneously. It stops meaning anything specific.

Medium · Hierarchy
Cards are whispers

Category tiles use #FAFBFF on #FFFFFF, a 1% tonal shift. The border carries all the work. On Retina at typical viewing distance, cards look like flat rectangles.

Medium · Spacing
No clean rhythm

Padding values slide between 12/14/16/20/24/32px with no base unit. Tabs sit tight against content; the greeting card breathes too much. Feels hand-placed, not systematic.

Medium · Components
3px radius everywhere

Global --radius: 3px flattens buttons, cards, inputs, badges and modals to the same shape. Nothing feels distinct. Buttons in particular want to be chunkier than their container.

Low · Voice
Serif headline + emoji CTAs

"Burning the midnight oil, Deena" is a lovely touch, but sits under a Lora heading and next to a ⚡ upgrade button. The tone is schizophrenic — part editorial, part Duolingo.

Low · Density
Screen has one job, feels like six

Greeting card + tabs + stats strip + Gmail nudge + suggestion chips + input + CTA + nudge all stacked. The actual primary action (search) is buried in the middle.

Low · Sidebar
Navy sidebar reads "admin tool"

The #060D1A sidebar is the most visually dominant element on every screen, but it carries the least-used nav. Consider inverting the emphasis.

02 — Principles

Four rules for every decision from here on.

Think of these as tiebreakers: when there's a choice between "calm" and "clever", pick calm.

01
Quiet surfaces, loud content

The UI shouldn't out-decorate the data. Warm paper background, hairline borders, no gradients. Color is reserved for signals — states, actions, alerts — never "decoration".

02
One system, one scale

One sans for UI, one serif for display, one mono for metadata. One 4px spacing base. One color ramp. If a token can't justify its existence, it goes.

03
Tools, not dashboards

Offerloop is a workbench for recruiting. Mono labels, keyboard shortcuts, tabular numbers, dense rows. It should feel like Linear / Notion / Pitch — not a marketing site in disguise.

04
Earn every emphasis

If everything is bold, nothing is. Use bold, color, size, or ornament for one thing per screen — the primary action or the primary insight. Everything else recedes.

03 — Type

Geist for everything, Fraunces for moments, Geist Mono for labels.

We drop Inter, Lora, Instrument Serif, DM Sans, Libre Baskerville, IBM Plex Mono and VanquishBold in favour of three families with clear jobs. Fraunces replaces Lora because its variable opsz axis gives the display headlines real presence without turning body UI into a novel.

04 — Color

Warm paper, one deep brand blue, amber accents kept rare.

Defined in oklch so lightness scales evenly across hues. The palette has three jobs: neutrals (the entire UI surface), brand (one action color, used sparingly), signals (states only). That's it.

Next

Suggested rollout.

Low-risk order — start with tokens, end with marketing.

Step 01
Tokens first

Replace index.css variables with this set. Keep Tailwind config; swap values. Ship behind a feature flag.

Step 02
Swap fonts

Remove 5 Google Font imports; load Geist + Fraunces + Geist Mono. Delete VanquishBold. Audit Lora references.

Step 03
Rebuild 6 primitives

Button, Input, Badge, Card, Tab, Row. Everything downstream reuses these. Delete one-off CSS.

Step 04
Marketing last

Landing follows product, not the other way. Apply tokens to landing once in-app screens settle.
