# P0 Email Quality Baseline — 2026-04-28

Baseline snapshot before shipping the email personalization overhaul.
All metrics below are **pre-deploy** values. Compare against these after 30 days
to measure impact.

## What shipped

| Change | Description |
|--------|-------------|
| P0: Personalization pipeline | Normalized profiles, 10 lead types, warmth tier upgrades |
| P1a: Subject specificity | Subjects must reference contact-specific proper nouns |
| P1b: Batch diversity | No duplicate subject prefixes (3-word) or openers (first sentence + 8-word prefix) |
| Cold email structural fix | 4-part structure for career_path/role_match; 3-part for general; anti-hallucination directive |
| Ask variety | LLM instructed to vary closing ask across batch; 4 variants provided |
| Opener diversity | LLM instructed to vary positioning sentence; 4 structural patterns; 8-word dedup |
| Firestore metadata | leadType, commonalityTypes, warmthTierFinal, wordCountFinal, leadHookUsedInBody |

## Firestore query filter

Only measure emails generated **on or after** 2026-04-28:

```
emailGeneratedAt >= 2026-04-28
```

Old emails do not have the new metadata fields. Do NOT backfill.

## Metrics to track (30-day window: 2026-04-28 to 2026-05-28)

### Email generation quality

| Metric | How to measure | Pre-deploy baseline | Target |
|--------|---------------|---------------------|--------|
| Lead type distribution | `leadType` field on contacts with `emailGeneratedAt >= 2026-04-28` | ~60-70% were `general` (old system had no lead detection) | <30% `general` |
| Quality gate pass rate | Count emails that passed `check_email_quality` on first attempt (no regen) | Not tracked before | >85% first-pass |
| Batch diversity regen rate | Count of `check_batch_diversity` triggering regeneration | Not tracked before | <15% of batches |
| Word count (cold tier) | `wordCountFinal` where `warmthTierFinal = cold` | 44-78 words (highly variable, many skeletal) | 60-90 words, no sub-55 |
| Subject specificity | Emails passing `subject_no_contact_noun` check on first attempt | Not tracked before | >90% first-pass |

## Revert path

If quality degrades or errors spike:

1. The personalization pipeline is isolated in `backend/app/utils/personalization.py`
2. `reply_generation.py` imports from personalization — revert to the pre-P0 `detect_commonality` path by:
   - Reverting the `batch_generate_emails` changes in `reply_generation.py`
   - Reverting the quality gate changes in `email_quality.py`
   - Reverting the Firestore metadata expansion in `emails.py`
3. All changes are on branch `feature/find-recommendation-cards`
4. Git revert: `git revert <merge-commit-sha>` on main after merge
