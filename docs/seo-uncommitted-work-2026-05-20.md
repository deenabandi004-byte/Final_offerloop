## Blockers — why this cannot ship yet

1. **Routes not registered.** `App.tsx` has no `<Route>` entries and no imports for
   any of the 6 wrapper components. Visiting `/salary/goldman-sachs` today hits the
   SPA 404 fallback, not `SalaryPage`. All 100 URLs are currently unreachable.

2. **Sitemap advertises dead URLs.** `sitemap.xml` already lists all 100 new URLs.
   If deployed before the routes exist, Googlebot crawls 100 soft-404s — actively
   harmful to domain quality.

3. **`generateMeta.ts` not updated.** The templates call
   `generateMeta('salary', …)`, `generateMeta('target-schools', …)`, etc., but
   `generateMeta.ts` contains none of those 6 cases. New pages will fall through to
   the default and ship with generic/wrong `<title>` and meta description.

4. **Name collisions to watch.** `App.tsx` already imports an unrelated `AgentPage`
   (the protected in-app `/agent` page) and renders `RecruitingTimelinePage` at the
   protected `/recruiting-timeline` route. Wiring the new public pages needs aliased
   imports to avoid clobbering these.

## Observation — sitemap domain

The sitemap (and `generate-sitemap.cjs`, `const BASE`) uses `https://www.offerloop.ai`
(www). `CLAUDE.md` names the canonical domain as `https://offerloop.ai` (non-www).
This is **pre-existing** — the committed sitemap at HEAD already used www for all
1,494 URLs — so it is not a regression from this work. But it is worth resolving:
the sitemap should list canonical (non-www) URLs.

## Recommended next steps (in order)

1. Add 6 lazy imports + 6 public `<Route>` entries to `App.tsx` (alias to avoid the
   `AgentPage` / `RecruitingTimelinePage` collisions).
2. Add the 6 missing cases to `generateMeta.ts`.
3. Smoke-test one URL per route type in the browser (verify render + `<title>`).
4. Only then commit — and commit `sitemap.xml` in the **same** commit as the routes,
   never before, so the sitemap never points at unreachable pages.
5. Decide the www vs non-www canonical and fix `generate-sitemap.cjs` `BASE`.
6. After deploy: resubmit the sitemap in Search Console; watch indexation of the new
   route types over 2–4 weeks.
